# 配置说明

## 简单回答：只需要 App ID

对于这个飞书文档插件，**你只需要获取 `App ID` 就可以了**！

## 为什么只需要 App ID？

### 客户端插件 vs 服务端插件

这个插件是**客户端插件**，运行在飞书文档的浏览器环境中：

- ✅ **使用客户端 API**：直接调用 `window.bitable` API
- ✅ **不需要服务端**：所有操作都在浏览器中完成
- ✅ **只需要 App ID**：用于标识应用和权限验证

### App Secret 什么时候需要？

`App Secret` 只在以下场景需要：

- ❌ **服务端 API 调用**：如果你的后端服务器需要主动调用飞书 API
- ❌ **Webhook 验证**：如果需要验证飞书发送的 webhook 请求
- ❌ **OAuth 授权**：如果需要服务端处理 OAuth 流程

**本项目不需要这些功能，所以不需要 App Secret！**

## 配置步骤

### 1. 获取 App ID

1. 访问 [飞书开放平台](https://open.feishu.cn/)
2. 创建企业自建应用
3. 在应用详情页面找到 **App ID**
4. 复制 App ID（格式类似：`cli_xxxxxxxxxxxxx`）

### 2. 更新 manifest.json

只需要替换 `app_id`：

```json
{
  "app_id": "cli_xxxxxxxxxxxxx",  // 替换为你的 App ID
  "app_name": "文档筛选条件获取插件",
  "scopes": [
    "bitable:app",
    "bitable:app:readonly"
  ]
}
```

### 3. 申请权限

在飞书开放平台申请以下权限：

- `bitable:app` - 基础权限
- `bitable:app:readonly` - 只读权限（必需）

## 完整配置清单

### 必需配置

- ✅ **App ID** - 在飞书开放平台获取
- ✅ **权限申请** - `bitable:app:readonly`

### 可选配置

- ⚪ **应用图标** - 在 manifest.json 中配置 `icon`
- ⚪ **应用描述** - 在 manifest.json 中配置 `description`
- ⚪ **环境变量** - 如果需要，可以配置 `FEISHU_APP_ID`

### 不需要的配置

- ❌ **App Secret** - 不需要
- ❌ **Webhook URL** - 不需要
- ❌ **事件订阅** - 不需要（使用客户端事件监听）

## 常见问题

### Q: 为什么文档里提到了 App Secret？

A: 之前的文档可能包含了服务端场景的说明。对于这个客户端插件，确实不需要 App Secret。

### Q: 如果以后需要服务端功能怎么办？

A: 如果将来需要服务端功能（比如保存数据到自己的服务器），那时再配置 App Secret 即可。

### Q: App ID 会泄露吗？

A: App ID 是公开的，可以放在前端代码中。只有 App Secret 需要保密。

## 总结

**一句话总结**：只需要 App ID，配置到 `manifest.json` 中，申请权限，就可以使用了！
